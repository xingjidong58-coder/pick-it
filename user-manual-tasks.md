# 项目“就它了”用户手动操作指南

## 概述
本文档列出开发过程中需要**用户（刘易豪）亲自操作**的环节。AI助手（星悸动2号）将提供详细指导，但以下环节需要用户手动执行。

## 设备与环境限制
- **当前已知限制**: 用户没有Mac电脑
- **影响**: 无法直接开发iOS原生应用，需要替代方案
- **解决方案选择**:
  1. **Android优先** - 先开发Android版本（可在Windows开发）
  2. **PWA/Web应用** - 开发渐进式Web应用（跨平台，无需应用商店）
  3. **Expo云构建** - 使用Expo的云服务构建iOS版本（可能需要付费）

## 需要用户手动操作的环节

### 第一阶段：环境配置与软件安装

#### 1. 开发环境安装（Windows）
- [ ] **Node.js安装**
  - 访问 https://nodejs.org 下载LTS版本
  - 安装时勾选“Add to PATH”选项
  - 验证: 命令行运行 `node --version` 和 `npm --version`

- [ ] **Java JDK安装** (Android开发必需)
  - 下载JDK 11或17 (https://adoptium.net/)
  - 设置JAVA_HOME环境变量

- [ ] **Android Studio安装**
  - 下载Android Studio (https://developer.android.com/studio)
  - 安装时选择标准安装，包含Android SDK
  - 配置Android SDK路径

- [ ] **React Native CLI安装**
  - 命令行运行: `npm install -g react-native-cli`

- [ ] **Git安装与配置**
  - 下载Git (https://git-scm.com/)
  - 配置用户名和邮箱

#### 2. 项目仓库与协作工具
- [ ] **GitHub账户创建** (如果没有)
  - 访问 https://github.com 注册账户
  - 创建新的代码仓库

- [ ] **开发工具选择**
  - VS Code安装与扩展配置
  - 或选择其他喜欢的代码编辑器

### 第二阶段：设计决策与内容准备

#### 3. 应用设计决策
- [ ] **颜色方案确定**
  - 主色调、辅助色、文字颜色
  - 品牌标识设计（Logo、应用图标）

- [ ] **UI/UX细节确认**
  - 页面布局、交互方式
  - 按钮样式、动画效果

- [ ] **菜系列表定义**
  - 确定10个左右的核心菜系
  - 示例: 川菜、湘菜、粤菜、鲁菜、日料、韩餐、西餐等

- [ ] **虚拟奖品清单**
  - 确定积分可兑换的奖品类型
  - 外卖红包面额、视频会员类型等

#### 4. 数据准备
- [ ] **餐厅/菜品数据收集** (MVP阶段)
  - 收集20-30个常见餐厅信息
  - 菜品名称、价格区间、菜系分类
  - 可手动创建Excel/CSV文件

- [ ] **推荐规则初步定义**
  - 权重分配: 价格、距离、评分、口味偏好
  - 轻度模式与重度模式的具体差异

### 第三阶段：测试与反馈

#### 5. 应用测试
- [ ] **Android设备准备**
  - 准备Android手机用于测试（至少一部）
  - 开启开发者选项和USB调试

- [ ] **功能测试参与**
  - 安装测试版本，体验完整流程
  - 记录遇到的问题和优化建议

- [ ] **用户反馈收集**
  - 邀请3-5位朋友试用
  - 收集他们的使用反馈

### 第四阶段：部署与发布

#### 6. 后端服务器配置
- [ ] **云服务商选择与注册**
  - 选择: 阿里云、腾讯云、AWS、GCP等
  - 注册账户并完成实名认证

- [ ] **服务器购买与配置**
  - 选择配置（建议: 2核4G，最低配置）
  - 选择操作系统（推荐: Ubuntu 20.04+）

- [ ] **域名购买与解析** (可选)
  - 购买域名（如: pickit.com 或相关域名）
  - 配置DNS解析到服务器IP

#### 7. 应用商店发布

##### Android发布 (Google Play)
- [ ] **Google开发者账户注册**
  - 访问 https://play.google.com/console
  - 支付一次性注册费（约25美元）

- [ ] **应用信息准备**
  - 应用名称、描述、截图
  - 隐私政策、服务条款

- [ ] **应用签名证书生成**
  - 生成密钥库文件 (keystore)
  - 妥善保管密钥密码（重要！丢失无法更新应用）

##### iOS发布解决方案 (没有Mac的选项)
由于没有Mac电脑，iOS发布有以下方案：

**方案A: 使用Expo云构建** (推荐)
- [ ] **注册Expo账户** (https://expo.dev)
- [ ] **配置Apple开发者账户** (年费99美元)
- [ ] **通过Expo Dashboard提交iOS构建**
- [ ] **使用Expo的应用商店提交服务**

**方案B: 第三方iOS构建服务**
- [ ] **选择服务商** (如: Codemagic, Bitrise等)
- [ ] **配置构建流水线**
- [ ] **远程管理证书和配置文件**

**方案C: 仅发布Android版本**
- [ ] **专注Android市场** (中国用户大部分使用Android)
- [ ] **后期考虑iOS版本** (当有Mac设备或预算时)

**方案D: 发布为PWA** (渐进式Web应用)
- [ ] **优化Web应用为PWA**
- [ ] **通过浏览器访问，可添加到主屏幕**
- [ ] **绕过应用商店审查**

### 第五阶段：运营与维护

#### 8. 运营准备
- [ ] **监控工具配置**
  - 应用性能监控 (如: Sentry)
  - 用户行为分析 (如: Google Analytics)

- [ ] **客服渠道建立**
  - 用户反馈收集方式
  - 问题响应机制

#### 9. 商业化准备
- [ ] **支付渠道对接** (如果需要)
  - 微信支付、支付宝商户申请
- [ ] **合作洽谈** (与外卖平台)
  - 联系美团、饿了么等平台谈合作

## 时间预估与分工

### AI助手负责的部分
1. **代码编写与架构设计**
2. **开发指导与环境配置指导**
3. **问题解决与调试帮助**
4. **文档编写与最佳实践建议**
5. **自动化脚本提供**

### 用户负责的部分
1. **软件安装与环境配置** (按指导手动操作)
2. **设计决策与内容准备** (提供方向和要求)
3. **测试反馈与需求调整** (实际使用并反馈)
4. **账户注册与支付** (需要身份验证的部分)
5. **最终发布操作** (应用商店提交等)

### 每周时间投入建议
- **用户**: 每天1-2小时，主要用于决策、测试、反馈
- **AI**: 全天候支持，根据用户进度提供指导

## 风险缓解措施

### 没有Mac的风险缓解
1. **Android优先**: 先确保Android版本完美
2. **Web备用**: 开发响应式Web版本作为备用
3. **云构建服务**: 使用Expo等服务的免费额度测试iOS可行性
4. **后期考虑**: 等项目有收入后再考虑购买Mac设备

### 技术复杂度风险
1. **简化MVP**: 初期使用基于规则的推荐，不用复杂算法
2. **模拟数据**: 初期使用本地模拟数据，不依赖外卖平台API
3. **分阶段开发**: 先核心功能，后增强功能

### 用户时间投入风险
1. **明确分工**: AI处理技术实现，用户处理决策和测试
2. **自动化脚本**: AI提供一键安装和配置脚本
3. **清晰文档**: 每个手动步骤都有详细指导
