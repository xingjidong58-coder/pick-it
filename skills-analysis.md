# 项目“就它了”技能需求分析

## 项目技术需求总结

### 核心功能模块
1. **移动端应用开发** (iOS/Android)
2. **后端API服务** (用户管理、推荐算法、积分系统)
3. **数据库设计与管理** (用户数据、推荐历史、积分记录)
4. **推荐算法实现** (个性化菜品推荐)
5. **第三方API集成** (外卖平台、支付接口)
6. **UI/UX设计** (用户友好的决策界面)

## 现有技能评估

### 已安装的技能（工作空间）
1. **frontend-design-3** - 前端UI设计（色彩、字体、排版规范）
   - **适用性**: 可用于移动端UI设计，有一定帮助
   - **限制**: 主要针对Web前端，移动端适配需额外知识

2. **coding-agent** - 代码编写助手（通用编程）
   - **适用性**: 通用代码编写，可用于前后端开发
   - **限制**: 需要具体的框架和语言指导

3. **security-auditor** - 安全审计（代码安全检查）
   - **适用性**: 代码安全审查，防止常见漏洞
   - **限制**: 非核心开发技能，但重要

4. **security-scanner** - 安全扫描（漏洞检测）
   - **适用性**: 安全测试，生产环境安全检查
   - **限制**: 非核心开发技能

5. **github** - GitHub操作（版本控制、CI/CD）
   - **适用性**: 代码版本管理、协作开发
   - **核心需求**: 必需，项目协作基础

6. **obsidian** - 文档管理（项目文档）
   - **适用性**: 项目文档管理、知识库
   - **核心需求**: 有用，非必需

7. **ontology** - 知识图谱（结构化数据建模）
   - **适用性**: 用户偏好数据建模、关系管理
   - **核心需求**: 对推荐系统数据建模非常有用

### 已安装的技能（全局技能目录）
8. **clawhub** - 技能管理（搜索、安装、更新）
   - **适用性**: 安装新技能的必需工具
   - **核心需求**: 必需

## 技能安装优先级规划

## 技能依赖关系分析

### 技能依赖图
```
用户需求 → 移动端界面 → react-native
                    ↘ 后端服务 → backend → database-design
                    ↘ 推荐算法 → what-to-eat → automl-skill (可选)
                    ↘ API集成 → api-integration
```

### 学习路径建议
1. **第一周**: react-native + backend 基础
2. **第二周**: database-design + api-integration 实现
3. **第三周**: what-to-eat 算法集成 + UI/UX优化
4. **第四周**: 测试、部署、算法优化

## 风险评估

### 高风险技能缺口
1. **外卖平台API集成**: 需要研究美团/淘宝/京东的开放API
2. **推荐算法准确性**: 初期可能依赖简单规则，需要逐步优化
3. **移动端性能**: React Native在复杂动画和性能方面的限制

### 缓解措施
1. **初期简化**: 使用模拟数据，不直接集成外卖平台API
2. **规则引擎先行**: 初期使用基于规则的推荐，逐步引入机器学习
3. **性能优化**: 使用原生模块优化关键性能路径
